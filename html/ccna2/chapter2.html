Chapter 2 - Switching Concepts

Frame Forwarding

Switching in Networking

The decision on how a switch forwards traffic is made based on the flow of that traffic.

Two terms associated with the frames entering and leaving an interface:

  • ingress
    • port where a frame enters the device
  • egress
    • port where frame use when leaving the device

The Switch Learn and Forward Method

  • Step 1: Learn - Examining the Source MAC Address
    • switch add source MAC address and incoming port to number to the table
    • if MAC address does exist in table, switch will refresh timer for that entry
    • if MAC address does exist but on different port —> refresh entry with current port

  • Step 2: Forward - Examining the Destination MAC Address
    • MAC is in table - done
    • MAC is not in table —> frame is forwarded to all port except incoming

Store-and-Forward Switching

  • Error checking
    • after receiving the entire frame on the ingress port, the switch compares the FCS of datagram with its own FCS calculations
    • error-free —> frame is forwarded

  • Automatic buffering
    • support any mix of Ethernet speeds
    • ex - ingress port is 100 Mbps, switch stores it into buffer, check FCS and forward to egress port with 1Gbps

Cut-Through Switching

  • not checking for errors
  • forward frame as soon as looking up the destination MAC address
  • fragment-free switching is C-T switching with better error checking with no increase in latency
