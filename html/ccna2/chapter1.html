Chapter 1 - Basic Device Configuration

Configure a Switch with Initial Settings

Switch Boot Sequence

After a Cisco switch is powered on, it goes through 5-step boot sequence

  • Step 1
    • switch load a power-on self-test POST program stored in ROM
    • POST checks the CPU subsystem
    • it test CPU, DRAM and the portion of the flash device that makes up the flash file system
  • Step 2
    • switch loads the boot loader SW
    • boot loader is small program stored in ROM, run immediately after POST is successful
  • Step 3
    • boot loader performs low-level CPU initialization
    • it initializes the CPU registers (registers control where a physical memory is mapped, the quantity of memory and its speed)
  • Step 4
    • boot loader initializes the flash file system on the system board
  • Step 5
    • finally the boot loader locates and loads a default IOS operating system SW image into memory and gives control of the switch over to the IOS

The boot system command

If the BOOT variable is not set, switch will load and execute the first executable file it can find while starting.

On Cisco switches (2960) the image file is normally contained in a directory that has the same name as the image file:

S1(config)# boot system flash:/c2960-lanbasek9-mz.150-2.SE/c2960-lanbasek9-mz.150-2.SE.bin

Then Cisco IOS commands initializes the interfaces from startup-config file (config.txt, located in flash)

show boot- see what the current IOS boot file is set to

Recovering from a System Crash

The boot loader provides access into the switch if the operating system cannot be used (missing or damaged system files).

The boot loader has a command line —> access to files in flash memory

he boot loader commands support initializing flash, formatting flash, installing a new IOS, changing the BOOT environment variable and recovery of lost or forgotten passwords.

Switch Management Access

Remote access - switch must be configured with an IP address and a subnet mask

Remote network - switch must have configured default gateway

Note: The switch may need to be configured for IPv6. For example, before you can configure IPv6 addressing on a Cisco Catalyst 2960 running IOS version 15.0, you will need to enter the global configuration command sdm prefer dual-ipv4-and-ipv6 default and then reload the switch.
